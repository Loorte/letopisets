# ИИ-агенты

Система использует Claude API (@anthropic-ai/sdk) для симуляции мира. Агенты работают по тиковой модели — каждый тик симуляции оркестратор определяет, какие агенты должны действовать.

## Архитектура

```
                    ┌──────────────────┐
                    │    Simulation     │
                    │    Engine         │
                    │    (BullMQ тик)   │
                    └────────┬─────────┘
                             │
                    ┌────────▼─────────┐
                    │   ОРКЕСТРАТОР     │
                    │   (Sonnet)        │
                    │   Решает кого     │
                    │   вызывать        │
                    └──┬──┬──┬──┬──┬───┘
                ┌──────┘  │  │  │  └──────┐
          ┌─────▼──┐ ┌────▼┐ │ ┌▼─────┐ ┌▼────────┐
          │Политика│ │Экон.│ │ │Воен. │ │Мировые  │
          │Sonnet  │ │Haiku│ │ │Sonnet│ │события  │
          └────────┘ └─────┘ │ └──────┘ │Sonnet   │
                        ┌────▼───┐      └─────────┘
                        │Бестиар.│
                        │Sonnet  │
                        └────┬───┘
                        ┌────▼───┐
                        │  NPC   │
                        │ Haiku  │
                        └────────┘
```

## Агенты

### Оркестратор

- **Модель**: Sonnet
- **Частота**: каждый тик
- **Роль**: анализирует текущее состояние мира и определяет, каких агентов вызвать и в каком порядке. Не все агенты работают каждый тик — это оптимизирует стоимость.
- **Файл**: `packages/server/src/agents/orchestrator.ts`

### Политика

- **Модель**: Sonnet
- **Частота**: каждый тик
- **Роль**: дипломатические отношения, альянсы, договоры, интриги, перевороты, смена правителей
- **Влияет на**: `states`, `diplomatic_relations`, `npcs` (правители), `world_events`
- **Промпт**: `packages/server/src/agents/prompts/politics.prompt.ts`

### Экономика

- **Модель**: Haiku (формульная логика, достаточно дешёвой модели)
- **Частота**: каждые 5 тиков
- **Роль**: торговые пути, производство, рыночные цены, экономический рост/упадок
- **Влияет на**: `burgs`, `trade_routes`, `states` (economy), `world_events`

### Военный

- **Модель**: Sonnet
- **Частота**: при активных конфликтах (diplomatic_relation = war)
- **Роль**: армии, битвы, осады, территориальные изменения, набор войск
- **Влияет на**: `states` (military, population), `burgs`, `npcs` (генералы), `world_events`

### Бестиарий

- **Модель**: Sonnet
- **Частота**: каждые 2-3 тика
- **Роль**: управление монстрами и существами мира
- **Действия**:
  - **Миграция**: стаи перемещаются между биомами по сезонам
  - **Нападения**: атаки на караваны, осады городов, похищение NPC
  - **Популяция**: размножение, вымирание от охотников и героев
  - **Разумные существа**: вампиры и демоны ведут интриги, заключают сделки
  - **Легенды**: уникальные существа порождают цепочки событий
  - **Экология**: баланс хищник-жертва, влияние на экономику
- **Влияет на**: `creatures`, `burgs`, `trade_routes`, `npcs`, `world_events`

### NPC

- **Модель**: Haiku (вызывается часто, нужна экономия)
- **Частота**: батчами по группам (по государству/роли)
- **Роль**: поведение персонажей, принятие решений, личные цели, сюжетные линии
- **Влияет на**: `npcs`, `world_events`

### Мировые события

- **Модель**: Sonnet
- **Частота**: каждые 3 тика
- **Роль**: случайные глобальные события — стихийные бедствия, эпидемии, открытия, праздники, магические явления
- **Влияет на**: все таблицы в зависимости от типа события

## Трёхуровневая память

Каждый агент имеет доступ к трём уровням памяти.

### Рабочая память (working)

Контекст текущего тика, передаётся в промпт агента:
- Номер текущего тика
- Краткая сводка состояния мира
- События за последние 5 тиков
- Активные ситуации (войны, кризисы, квесты)

### Долгосрочная память (long_term)

Хранится в таблице `agent_memories`, извлекается по релевантности:
- Top-20 записей по `importance` + свежести
- Фильтрация по `agent_type` и связанным сущностям
- Каждый вызов агента может создать новые записи

### Рефлексивная память (reflective)

Периодические сводки, создаваемые агентами:
- Каждые N тиков агент генерирует обзор трендов и прогнозов
- Высокий `importance` — приоритет при извлечении
- Обеспечивает непрерывность на длинных симуляциях

## Инструменты агентов

Агенты могут вызывать структурированные инструменты (tools) для взаимодействия с миром.

| Инструмент | Описание |
|-----------|---------|
| `query_world_state` | Запрос текущего состояния: государства, города, NPC, события |
| `modify_entity` | Изменение сущности с указанием причины |
| `create_event` | Создание нового мирового события |
| `diplomacy_action` | Дипломатическое действие (союз, война, договор) |

Все инструменты определены в `packages/server/src/agents/tools/`.

## Управление стоимостью

| Стратегия | Описание |
|----------|---------|
| Выборочный запуск | Оркестратор решает, кого вызывать — не все агенты работают каждый тик |
| Разные модели | Haiku для рутины (~$0.25/M input), Sonnet для решений (~$3/M input) |
| Лимит бюджета | ~$0.01-0.03 на тик |
| Батчевая обработка | NPC обрабатываются группами, а не по одному |
| Кеширование | Если состояние не изменилось — пропуск вызова агента |

### Оценка стоимости

- ~2 000-5 000 input-токенов на тик (все агенты суммарно)
- ~500-1 500 output-токенов на тик
- 100 тиков/день: **~$1-3/день**
- Ежемесячно: **~$30-90** при активной симуляции

## Формат ответа агента

Каждый агент возвращает структурированный JSON, который валидируется через Zod:

```json
{
  "actions": [
    {
      "type": "modify_entity",
      "entityType": "state",
      "entityId": "uuid-...",
      "changes": { "stability": 35 },
      "reason": "Народные волнения из-за высоких налогов"
    },
    {
      "type": "create_event",
      "eventType": "political",
      "title": "Восстание в Эльдории",
      "description": "Крестьяне подняли мятеж...",
      "severity": "major",
      "affectedEntities": ["uuid-state", "uuid-burg"]
    }
  ],
  "memory": "Эльдория нестабильна уже 3 тика. Следить за развитием."
}
```
